## 6. How to use this output

1. Seed code (`packages/db/src/seed-historical.ts`) reads `canonical-data.json` instead of opening XLSX files at runtime where possible.
2. For row-level data, the JSON contains every cell with formula + cached value. Use the cached value; if `needsRecache` is true, the seed must compute itself.
3. Anomalies should be reviewed when the seed produces unexpected row counts.
4. Regenerate with `bun run sot:extract` whenever `source-of-truth/` changes (write fresh sha256 and commit).
